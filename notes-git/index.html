git笔记

作用

分布式版本管理器

git服务器

git本地服务器与远程服务器

  • git本地服务器：是搭建在本地的git环境(安装在本地的git);

    ​ 所有的git add 与git commit提交都是在本地服务器完成的;

  • git远程服务器：是指非本地的git服务器

    ​ git push就是把本地信息推送到远程服务器上。

安装git

  • windows

    去官网下载

  • Linux

yum install -y git

配置用户

创建仓库

  • 鼠标右键新建一个目录
  • Linux命令mkdir创建目录

初始化仓库

创建好仓库后,需要先进入仓库git init初始化仓库,也就是让git监控这个目录;

添加与提交

提交与执行都是在本地服务器完成的，就是把信息丢到本地仓库。
git add ... 添加
git add -A/--all 添加所有的变化
git add -u 添加被修改和被删除的文件，不包括新文件
git add . 添加新文件和被修改的文件，不包括被删除的文件
git commit -m "版本1" 提交
-m是指：message

查看

  • 查看状态
    git status

  • 查看修改的内容
    git diff readme.txt

  • 查看版本
    git log
    git log --pretty=oneline 排成一行显示
    git log --graph --pretty=oneline --abbrev-commit 排成一行，简化ID,并显示合并相关的信息

  • 查看历史版本操作记录

git reflog

  • 查看远程服务器信息

git remote 查看远程服务器的信息
输出的内容是服务器的名字;
如果没有指定，远程服务器的名字默认为:origin;
git remote -v 详细查看服务器的信息
如果有push(推送)权限,就会显示连接的远程服务器freth和push的url

版本操作

  • 回退一个版本

git reset --hard HEAD^

  • 回退到指定版本
    git reset --hard 版本号/版本ID
    版本号可以用git reflog或者git log查看

撤消操作

  • 撤消修改
    git checkout -- readme.txt
    针对修改了,却没有添加(git add)到分枝的操作
  • 撤消添加
    git reset HEAD -- readme.txt
    撤消添加(git add)到分枝的操作

远程服务器

  • 生成秘钥
    ssh-keygen -t rsa -C "youremail@example.com"
    生成两个文件一个是id_rsa(私钥)和id_rsa.pub(公钥)
    首次连接github要生成秘钥，并且要在github帐号添加相关的公钥

  • 添加关联远程仓库
    git remote add origin git@github.com:Fengdo/learnGit.git
    git remote add myServer git@gitee.com:Fengdo/learnGit.git
    让本地与远程仓库关联起来，实现本地与远程仓库同步。
    git 可以同时关联多个远程服务器，并可以选择性的提交数据到指定服务器
    添加关联时，可以给服务器取个名字。比如：origin和myServer就是自定义的名字
    remote是服务器的意思;
    origin是服务器的名字;

  • 推送到远程仓库
    git push -u origin master 把主分枝推送到origin服务器
    git push gitee dev 把dev分枝推送到gitee服务器
    首次推送要加-u,后续的不用
    origin是指服务器的名字
    master是指主分枝

  • 克隆项目
    git clone git@github.com:Fengdo/learnGit.git

  • 克隆别人的项目进行开发
    先克隆，后取消别人的远程关联url。

  • 修改克隆项目url
    进入隐藏目录.git修改config文件的url

  • 推送(push)

    git push origin master推送==主分枝==到服务器

    git push origin <分支名称>推送==指定分枝==到服务器

  • 删除远程github关联
    git remote rm origin

分支

  • 了解分支
    主分支与其它分支

  • 指针指向
    master指向：主分支
    dev指向：其它分支
    HEAD指向：当前分支

    ==注意==：如果把HEAD指向dev，那么dev所指向的分支就是当前的分支。这并不影响主要分支的内容。

  • 察看分支
    git branch
    列出所有分支而且当前分支前面会标一个*号。

  • 切换分支
    git checkout <name> 切换到主分支

  • 创建分支
    git branch <name>
    增加一个dev指向

  • 合并分支

把master指向dev提交的内容

  • 合并分支模式
    Fast-forward模式，删除分支后会丢失分支信息
    禁用Fast-forward模式，git merge --no-ff -m "merge with no-ff" dev
    –no-ff 参数表示禁用Fast-forward
    -m表示message
  • 解决冲突
    当分支存在冲突时，要先手动解决冲突，然后重新提交
    当冲突时，合并的内容冲突部分会被<<<<<<< HEAD ======= >>>>>>>标出
    git log --graph可以查看分支合并图
  • 删除分支

删除dev的指向
