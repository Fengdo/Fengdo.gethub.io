初始化

安装完毕后，新建一个空目录然后初始化
非空目录会报错，只能用空目录
初始化的过程中会下载一些后面要使用的目录和文件

  1. hexo init 根目录
  2. cd 根目录
  3. npm install //下载package.json的依赖包

初始化完成后会生成以下文件和目录

  1. _config.yml:网站配置文件
  2. package.json:程序应用信息
  3. scaffolds:模板文件夹 //生成的文章md文件所包函的内容,比如page页面和post页面的头部内容
  4. source:资源存放文件夹
  5. themes:主题文件夹

网站配置

  1. title 网站标题
  2. subtitle 副标题
  3. description 网站描述 //用于seo简短描述网站
  4. author 作者
  5. language 语言 //zh
  6. timezone 时区 //默认为本地电脑的时间
  7. url 域名 //远程绑定的域名
  8. root 根目录 / //子目录要修改成子目录的目录名;例如：/blog/
  9. permalink 文章永久链接格式 例如: :year/:month/:day/:title/
  10. permalink_defaults 永久链接默认值

清理缓存

清理缓存一般用于修改了配置或者模板文件而远程不生效时

生成静态的文件

比如：css,html等

可选指令
-d 文件生成后立即部署网站
-w 监听文件变动 //文件一有变动立马生成静态文件

调试【本地服务器】

本地服务器用于在发布前本地预览和修改代码进行调试
运行后会出现一个本地服务器链接:localhost:4000

可选指令
-p 重设端口
-s 只使用静态文件
-l 启动日志
