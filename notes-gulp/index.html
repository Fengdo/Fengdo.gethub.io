Gulp 4.0

说明

gulp升级到4.0,和以前的3.0完全不同了。
gulp把安装包从npm中分离到了gulp-cli

组成

  • gulp-cli包
  • Local version

gulp版本查询

gulp -v 或者 gulp --version
使用后，能看到两个版本的信息

删除gulp

npm rm --global gulp

安装gulp-cli包

npm install --global gulp-cli

gulp的安装步骤：

  1. 删除旧的gulp版本npm rm --global gulp
  2. 查看npm版本npm --version,确保都是最新版本,如果不是就升级npm install -g npm,目前是6.6版本
  3. 查看npx的版本信息npx -v
  4. 安装gulp-clinpm install --global gulp-cli
  5. 创建项目目录npx mkdirp 项目名
  6. 创建package.json配置文件 npm init
  7. 局部安装gulp npm install --save-dev gulp
  8. 查看gulp的版本信息gulp -v,应该能看到CLI version 2.0.1和 Local version 4.0.0
  9. 创建gulpfile.js(忽略大小写)文件 touch gulpfile.js

文件说明

  • package.json npm项目插件依赖的配置文件
  • package-lock.json 本地的依赖文件(这个不用管它)
  • gulpfile.js 自定义的流程;比如：创建目录树，编辑文件，打包上传

测试运行

gulpfile.js

gulpfile.js能包括多个js文件，只需要引入就行了
比如：在gulpfile.js个引入 index.js 就能把index.js变成它的一部分和less一样

任务说明

每一个gulp任务都是异步执行的，分公有和私有。

  • 公有 能直接调用的(有exports的)
  • 私有 相当于局部函数，如果没有设置出口(exports)是不会被调用的
