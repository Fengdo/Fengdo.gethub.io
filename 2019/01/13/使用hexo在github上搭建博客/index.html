使用hexo在github上搭建博客

本地安装hexo

1.0 安装hexo
1.1 创建一个[github的用户名].github.io的根目录
1.2 hexo init初始化这个根目录

创建仓库

在github上创建一个新的仓库，命名为[github的用户名].github.io不生成解释文件

配置网站

修改网站信息

修改网站url

修改主题

修改主题前要先把下载好的主题放在themes目录下
回到配置文件，把theme改为主题的名字

修改git远程服务关联

打开_config.yml修改deploy下的type: git;
在type下增加一行repo: 仓库地址 //git@github.com:用户名/用户名.github.io.git;
再增加一行branch: master;

hexo发布到远程

hexo d -g把本地代码发布的远程
进入仓库，找到设置，打开githubPage设置分支为主分枝

绑定自已的域名

进入仓库->设置,找到githubPage下面的自定义域名,修改为自己的域名;
进入域名商提供的域名解析服务，添加CNAME解析两条:记录值为github提供的域名二级域名,记录为@;第二条记录值相同，记录为www;

增加CNAME文件

不增加CNAME文件会导致每次发布页面时就要重新绑定自定义域名

在根目录的source目录下新建一个CNAME文件，没有后辍，然后把你的自定义域名写在里面
