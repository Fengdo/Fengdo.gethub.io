V2ray的搭建

时间校正

v2ray要求服务器时间和本地一致

查看服务器时间

xshell进入服务器

修改时间

date –set

date --set=可以修改时间[cetos7x64有效]，但是debain9修改了后并没有用;

tzselect

另一种方式就是tzselect命令

输入后的步骤

  • 选：Asia
  • 选：China
  • 选:ShangHai【上海或北京】
  • 选：yes

修改后，还是无法生效，系统提示写入全局变量

vi /etc/profile

在最后添加一行：
export TZ='Asia/Shanghai'

更新全局变量
source /etc/profile

服务端配置

  • 服务 VMess
  • 点击uuid重新生成
  • alertID任意
  • 服务的端口自选，但是不要太小防止冲突
  • 动态端口和mKCP协议不打勾
  • 传输层加密不选

客户端配置

  • 服务器IP
  • 本地端口任意，但不要太小防止冲突
  • 选择服务1:vmess
  • 选择用户【自动生成的】
  • 服务器端口【与服务端配置的要一致】

下载客户端

打开这个链接，选择合适的。

配置文件

远程

  • 回到xshell服务器界面，
  • 把服务端生成的的json文件复制到/etc/v2ray/config.json 配置文件
  • systemctl start v2ray 启动v2ray
  • service v2ray status 查看状态，是否安装成功[active,runing为成功，如果不成功就再修改服务端的配置文件]

本地

  • 打开本地客户端的json文件，把生成的客户端配置替换原来的内容
  • 运行v2ray.exe

设置本地socks代理

v2ray和ssr不一样，它默认不是全局代理的。如果没有设置本地代理服务，根本就没有作用[坑了我好久]

firefox自带socks5代理，chrome可以下载Falcon Proxy插件

【firefox】

  • 选项
  • 网络设置
  • 手动代理配置
  • socks主机填入：127.0.0.1 端口：你自定义的[本地端口]

【chrome】

Falcon Proxy插件，填写的内容
配置名称：自定义
代理类型: socks5
代理地址：127.0.0.1,也就是本地
代理端口: 客户端配置时填写的那个
